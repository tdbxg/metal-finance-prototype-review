# 金属加工订单财务系统原型评审说明

## 打开方式

把整个 `metal_finance_prototype` 文件夹发给对方，对方双击 `index.html` 即可打开。

如果浏览器限制本地文件功能，可在该目录下启动静态服务：

```bash
python3 -m http.server 19180 --bind 127.0.0.1
```

然后访问：

```text
http://127.0.0.1:19180/
```

## 当前原型重点

这个原型不是最终系统，主要用于确认业务流程是否正确：

- 订单利润是否围绕单个加工订单核算
- 报价核算字段是否符合钣金/CNC/金属加工实际报价
- 收款、定金、尾款、应收提醒是否够用
- 材料、外协、物流、表面处理等成本归集是否合理
- 国内订单的增值税发票、对公收款、物流费用、银行手续费是否需要补充字段
- 拍照/上传发票、银行回单后写入财务数据的流程是否自然
- Excel 批量导入订单、应付、库存资料的模板字段是否够用

## 请评审的人重点反馈

### 老板/管理层

- 首页 KPI 是否能回答“这个月赚了多少钱”
- 是否能看清每个客户、每个订单赚不赚钱
- 是否还需要现金流、应收逾期、订单风险提醒

### 财务

- 应收应付字段是否够用
- 定金、尾款、发票状态、收款方式是否符合实际
- 是否需要凭证、科目、税率、开票、对账、反结账
- 票据识别后是否需要复核/审批再入账

### 业务

- 报价核算字段是否符合日常报价
- 阶梯报价数量是否合理
- 是否需要报价单模板、报价历史、报价成功率
- 是否需要客户图纸、询价记录、邮件/WhatsApp 记录

### 采购/外协

- 材料、表面处理、CNC 外协、物流费用分类是否够细
- 是否需要供应商报价比价、采购订单、到货、入库、付款联动
- 外协加工是否需要进度、质检、返工、扣款

### 生产/仓库

- 材料库存字段是否够用：材质、厚度、规格、批次、边角料、损耗率
- 工序是否需要更细：激光、折弯、焊接、打磨、攻牙、压铆、装配
- 是否需要工时采集、设备折旧分摊、工序进度

## 现在还没有做的内容

- 没有真实数据库，刷新后新增数据不会永久保存
- 票据识别是模拟 OCR，尚未接真实发票/OCR/银行流水接口
- 没有登录、角色权限、审计日志
- 没有真实开票、税务、凭证、会计科目
- 没有多人协作和服务器部署
- 没有图纸文件管理和报价单 PDF 输出

## 建议下一步补充

1. 浏览器本地保存，先让试用数据刷新不丢。
2. 新增订单录入和报价单保存。
3. 新增客户、供应商、材料基础资料。
4. 把票据识别结果做成“待复核”列表。
5. 增加 Excel 导入后的字段校验、重复订单检查、错误行提示。
6. 增加权限角色：老板、财务、业务、采购、生产、仓库。
7. 评估是否用 ERPNext 做正式底座。
